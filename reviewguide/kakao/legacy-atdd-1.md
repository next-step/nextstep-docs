## 리뷰 철학
- "왜 그렇게 작성했나요?" 처럼 질문하며 AI 활용으로 사고 없이 구현하는 것을 방지합니다.
- 올바른 테스트 구현 보다는 "왜 그렇게 구현했는지"에 초점을 맞춰 리뷰합니다.

## 학습 목표
1. 코드 작성 전 프로젝트 분석 및 비즈니스 로직 파악
2. 사용자 관점의 행위 중심 시나리오 설계
3. 목적에 맞는 테스트 구현
4. 효과적인 클로드 코드 활용

## 리뷰 포인트

### 1. 프로젝트 분석
- "테스트 작성 전에 어떻게 프로젝트를 탐색하셨나요?"
- "비즈니스 로직을 이해하기 위해 어떻게 접근하셨나요?"
- "클로드 코드에게 어떤 질문을 했나요?"

### 2. 우선순위 판단
- "어떤 시나리오 후보들을 고려하셨나요?"
- "왜 이 시나리오를 선택하셨나요?"
- "단순 조회 API를 제외한 이유가 있나요?"

### 3. 시나리오 설계
- "시나리오의 범위를 어떻게 정하셨나요?"
- "실패 케이스는 왜 중요한가요?"
- "2번째 시도의 실패로 무엇을 검증하려고 하셨나요?"

### 4. 테스트 데이터
- "초기 데이터를 SQL로 준비한 이유는? API와 비교해보셨나요?"
- "테스트 실행 순서가 바뀌어도 성공하나요?"

### 5. 검증 전략
- "`.statusCode(200)`만으로 충분한가요? 재고 감소를 어떻게 확인하나요?"
- "Repository로 직접 조회하는데, API만으로는 불가능했나요?"
- "재고 1개에 2번 선물 시도하면? 2번째 실패가 무엇을 증명하나요?"

### 6. AI 활용
